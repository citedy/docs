# MCP Integration

Citedy exposes a [Model Context Protocol](https://modelcontextprotocol.io/) (MCP) server that lets any MCP-compatible AI client use Citedy tools natively.

## Connecting

### Claude Desktop / Claude Code

Add to your MCP config (`~/.claude/claude_desktop_config.json` or project `.mcp.json`):

```json
{
  "mcpServers": {
    "citedy": {
      "url": "https://www.citedy.com/mcp",
      "headers": {
        "Authorization": "Bearer YOUR_API_KEY"
      }
    }
  }
}
```

### Cursor / Other MCP Clients

Use the Streamable HTTP transport:

- **URL**: `https://www.citedy.com/mcp`
- **Auth**: `Authorization: Bearer YOUR_API_KEY`

## How It Works

The MCP server wraps the Agent API. Each MCP tool maps to one or more API endpoints. The server handles:

- Authentication via your API key
- Input validation with Zod schemas
- Structured response envelopes with status, operation hints, and error details
- Automatic polling hints for async operations (video generation, ingestion, scouts)

## Response Envelope

Every tool returns a structured envelope:

```json
{
  "tool": "article.generate",
  "status": "completed",
  "result": { ... },
  "meta": {
    "request_id": "uuid",
    "endpoint": "POST /api/agent/autopilot",
    "http_status": 200
  }
}
```

For async operations, the envelope includes polling instructions:

```json
{
  "status": "processing",
  "operation": {
    "id": "uuid",
    "poll_tool": "shorts.get",
    "poll_input": { "id": "uuid" }
  }
}
```
